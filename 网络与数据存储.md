# 吃了么 - 网络与数据存储

## 一、数据架构设计

### 1.1 整体架构

```
用户设备
   │
   ├─ 本地存储 (LocalStorage/IndexedDB)
   │   ├─ 游戏进度
   │   ├─ 用户设置
   │   └─ 缓存数据
   │
   ├─ 微信小游戏框架
   │   ├─ 微信登录
   │   ├─ 分享功能
   │   ├─ 支付功能
   │   └─ 广告功能
   │
   └─ 游戏服务器 (可选)
       ├─ 用户数据同步
       ├─ 排行榜
       ├─ 活动数据
       └─ 防作弊验证
```

### 1.2 数据分类

| 数据类型 | 存储位置 | 同步策略 | 重要性 |
|---------|---------|---------|--------|
| 用户基础信息 | 本地+云端 | 实时同步 | 高 |
| 游戏进度 | 本地+云端 | 关卡结束时同步 | 高 |
| 金币/道具 | 本地+云端 | 变化时同步 | 高 |
| 设置项 | 本地 | 本地存储 | 中 |
| 排行榜 | 云端 | 实时获取 | 中 |
| 成就徽章 | 本地+云端 | 解锁时同步 | 中 |
| 缓存资源 | 本地 | 本地存储 | 低 |

## 二、本地存储设计

### 2.1 存储方案选择

**微信小游戏环境：**
- **主存储**：`wx.getStorageSync` / `wx.setStorageSync`
- **容量限制**：单个 key 最大 1MB，总容量 10MB
- **备选**：IndexedDB（大量数据）

## 三、云端数据同步

### 3.1 同步策略

**实时同步（高优先级）：**
- 金币变化
- 道具使用
- 关卡完成
- 成就解锁

**延迟同步（中优先级）：**
- 游戏设置
- 统计数据
- 排行榜分数

**本地优先（低优先级）：**
- 缓存资源
- 临时数据

### 3.2 同步机制

**冲突解决策略：**
```
1. 时间戳优先：以时间最新的数据为准
2. 数值累加：金币、分数等可累加数据
3. 云端优先：关键进度以云端为准
4. 手动合并：冲突严重时提示用户选择
```

**断网处理：**
```
1. 本地继续游戏
2. 标记待同步数据
3. 网络恢复后自动同步
4. 同步失败时重试3次
5. 多次失败后提示用户
```

### 3.3 API 接口设计

**用户相关：**
```
POST /api/user/login
- 参数: { code: 微信登录code }
- 返回: { token, userId, userData }

GET /api/user/profile
- 参数: { userId }
- 返回: { userInfo, stats }

POST /api/user/update
- 参数: { userId, data }
- 返回: { success, message }
```

**游戏进度相关：**
```
GET /api/progress/{userId}
- 返回: { gameProgress }

POST /api/progress/save
- 参数: { userId, progress }
- 返回: { success, synced }

POST /api/progress/backup
- 参数: { userId, backupData }
- 返回: { backupId }
```

**排行榜相关：**
```
GET /api/rank/global
- 参数: { type: score/combo/time, page, limit }
- 返回: { ranks: [{ rank, userId, score, avatar }] }

GET /api/rank/friends
- 参数: { userId, type }
- 返回: { ranks }

POST /api/rank/submit
- 参数: { userId, levelId, score, time }
- 返回: { rank, bestScore }
```

## 五、数据安全

### 5.1 数据加密

**本地存储加密：**
```javascript
// 简单加密（可选）
const encrypt = (data) => {
  const json = JSON.stringify(data);
  return btoa(json); // Base64编码
};

const decrypt = (encoded) => {
  const json = atob(encoded);
  return JSON.parse(json);
};
```

**传输加密：**
- 使用 HTTPS 协议
- 敏感数据额外加密

### 5.2 数据备份

**自动备份：**
- 每日自动备份到云端
- 关卡完成时即时备份
- 保留最近10个备份点

**手动备份：**
- 用户可手动触发备份
- 备份码/二维码恢复

## 七、错误处理

### 7.1 存储错误

| 错误类型 | 处理方案 |
|---------|---------|
| 存储空间不足 | 清理缓存，提示用户 |
| 写入失败 | 重试3次，记录日志 |
| 读取失败 | 使用默认数据，提示错误 |
| 数据损坏 | 尝试恢复备份，否则重置 |

### 7.2 同步错误

| 错误类型 | 处理方案 |
|---------|---------|
| 网络错误 | 标记待同步，稍后重试 |
| 服务器错误 | 稍后重试，联系客服 |
| 数据冲突 | 按策略解决，记录日志 |
| 认证失效 | 重新登录 |

## 八、数据迁移

### 8.1 版本升级

```javascript
// 数据迁移示例
const migrateData = (oldData, oldVersion, newVersion) => {
  let data = oldData;
  
  if (oldVersion < "1.1") {
    // 1.0 -> 1.1 迁移
    data = migrateToV1_1(data);
  }
  
  if (oldVersion < "1.2") {
    // 1.1 -> 1.2 迁移
    data = migrateToV1_2(data);
  }
  
  return data;
};
```
