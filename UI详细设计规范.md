# 吃了么 - UI详细设计规范

## 一、设计原则与规范

### 1.1 设计理念
- **简洁清晰**：信息层级分明，重点突出
- **一致性**：全游戏UI风格统一
- **反馈及时**：用户操作有明确反馈
- **易于操作**：按钮大小适合移动端触控

### 1.2 视觉风格
- **整体风格**：卡通Q版，温馨治愈
- **圆角设计**：所有按钮和卡片使用圆角（8-16px）
- **阴影运用**：轻微投影增加层次感
- **动效原则**：弹性动画，反馈及时

## 二、字体规范

### 2.1 字体选择
- **中文字体**：思源黑体 / 苹方
- **英文字体**：Arial / Roboto
- **数字字体**：DIN Alternate Bold（分数、时间）
- **备用字体**：系统默认无衬线字体

### 2.2 字号规范

| 级别 | 用途 | 字号 | 字重 | 行高 |
|------|------|------|------|------|
| H1 | 页面大标题 | 48px | Bold | 1.2 |
| H2 | 章节标题 | 36px | Bold | 1.3 |
| H3 | 卡片标题 | 28px | Medium | 1.4 |
| H4 | 小标题 | 24px | Medium | 1.4 |
| Body | 正文内容 | 20px | Regular | 1.6 |
| Small | 辅助文字 | 16px | Regular | 1.5 |
| Caption | 说明文字 | 14px | Light | 1.4 |
| Number | 数字显示 | 32-64px | Bold | 1.0 |

### 2.3 字体颜色
- **主要文字**：#2F3542（黑色）
- **次要文字**：#57606F（深灰）
- **辅助文字**：#A4B0BE（灰色）
- **反白文字**：#FFFFFF（白色，用于深色背景）
- **链接文字**：#54A0FF（蓝色）
- **警示文字**：#FF6B6B（红色）

## 三、间距与布局

### 3.1 间距系统

**基础单位：4px**

| 名称 | 数值 | 用途 |
|------|------|------|
| XS | 4px | 图标内边距、紧凑间距 |
| SM | 8px | 小间距、按钮内边距 |
| MD | 16px | 标准间距、卡片内边距 |
| LG | 24px | 大间距、区块分隔 |
| XL | 32px | 页面边距、大区块 |
| XXL | 48px | 大模块间距 |

### 3.2 安全区域
- **顶部安全区**：iPhone刘海屏 44px / 其他 24px
- **底部安全区**：iPhone 34px / 其他 20px
- **左右边距**：24px（标准）/ 16px（紧凑）

### 3.3 栅格系统
- **设计画布**：750×1334px（iPhone标准）
- **列数**：4列（移动端适配）
- **列间距**：16px
- **内容区宽度**：750 - 48 = 702px

## 四、组件规范

**主按钮（Primary Button）：**
```
高度：88px（大）/ 72px（标准）/ 56px（小）
圆角：44px（圆形）/ 16px（圆角矩形）

文字：白色，28px，Bold
阴影：0 4px 12px rgba(255, 159, 67, 0.4)
内边距：左右48px

状态：
- 正常：#FF9F43
- 按下：#E67E22（变暗15%）
- 禁用：#DCDDE1（灰色）
```

**次按钮（Secondary Button）：**

圆角：16px
背景：透明
边框：2px solid #FF9F43
文字：#FF9F43，24px，Medium

状态：
- 正常：透明背景
- 按下：#FF9F43 10%透明度填充
```

**图标按钮（Icon Button）：**
```

状态：

### 4.2 卡片规范

## 六、动画规范

### 6.1 动画时长

| 动画类型 | 时长 | 缓动函数 |
|---------|------|---------|
| 微交互 | 0.15s | ease-out |
| 标准过渡 | 0.3s | ease-out |
| 页面切换 | 0.4s | ease-in-out |
| 弹窗出现 | 0.3s | spring |
| 加载动画 | 1.0s | linear |
| 庆祝动画 | 0.8s | ease-out |

### 6.2 常用动画效果

**按钮按下：**
```
transform: scale(0.95)
opacity: 0.9
duration: 0.1s
easing: ease-out
```

**卡片出现：**
```
transform: translateY(20px) → translateY(0)
opacity: 0 → 1
duration: 0.3s
easing: ease-out
```

**弹窗出现：**
```
transform: scale(0.8) → scale(1)
opacity: 0 → 1
duration: 0.3s
easing: cubic-bezier(0.175, 0.885, 0.32, 1.275) (spring)
```

**数字滚动：**
```
transform: translateY
steps: 10帧
duration: 0.5s
easing: ease-out
```

**摇晃提示：**
```
transform: rotate(-5deg) → rotate(5deg) → rotate(0)
duration: 0.4s
easing: ease-in-out
```

### 6.3 加载动画

**标准加载：**
```
类型：圆形旋转
尺寸：64×64px
颜色：主色
线宽：4px
动画：旋转360度，1s linear infinite
```

**骨架屏：**
```
背景：#F5F6FA
闪光：渐变从左到右
动画：位置移动，1.5s ease-in-out infinite
```

## 七、各页面UI详细规范

### 7.1 初始页面（主菜单）

**布局结构：**
```
顶部：Logo(120×120px) + 设置按钮 + 声音按钮
状态区：当前章节 + 总进度条(高度16px)
章节区：水平滑动，卡片尺寸200×280px
关卡区：网格布局，4列，卡片140×160px
信息区：分数、金币、徽章(横向排列)
底部：开始游戏按钮(大) + 每日任务 + 商城
```

### 7.2 游戏页面

**HUD布局：**
```
左上角：暂停按钮(64×64px)
中上：关卡名称(24px)
右上：分数显示(48px Bold)

中右：连击数(36px，金色)
中下：蓄力条(宽度60%，高度20px，圆角10px)

底部：道具栏，高度120px，3个槽位
```

### 7.3 结算页面

**通关布局：**
```
顶部：庆祝标题(48px) + 星级(96px星星)
中部：数据区(4行，左标签右数值)
奖励区：金币+徽章+道具(横向卡片)
底部：3个按钮(下一关、重玩、返回)
```
