# 吃了么 - 游戏重构版本

## 重构概述

本项目已从单一文件架构重构为模块化架构，增强了扩展性和可维护性。

## 新架构特点

### 1. 模块化设计
- **Core**: 核心引擎和基础类
- **Scenes**: 不同游戏场景
- **Entities**: 游戏实体对象
- **Services**: 服务类（API等）
- **Config**: 游戏配置文件
- **Systems**: 游戏系统（物理、渲染等）

### 2. 扩展性增强
- 配置驱动的游戏参数
- 插件式实体系统
- 模块化的场景管理
- 可扩展的API架构

### 3. 文件结构
```
frontend/
├── js/
│   ├── config/
│   │   └── GameConfig.js          # 游戏配置
│   ├── core/                     # 核心引擎
│   │   ├── GameEngine.js         # 游戏引擎
│   │   ├── SceneManager.js       # 场景管理器
│   │   ├── Scene.js              # 基础场景类
│   │   └── GameState.js          # 游戏状态管理
│   ├── entities/                 # 游戏实体
│   │   ├── Player.js             # 玩家类
│   │   ├── Platform.js           # 平台类
│   │   └── Obstacle.js           # 障碍物类
│   ├── scenes/                   # 场景
│   │   ├── StartScene.js         # 开始场景
│   │   ├── LevelSelectScene.js   # 关卡选择场景
│   │   └── GameScene.js          # 游戏场景
│   ├── services/                 # 服务
│   │   └── APIService.js         # API服务
│   └── main.js                   # 主入口文件
└── index.html
```

### 4. 后端扩展
- 多章节支持
- 自定义关卡系统
- 成就系统
- 社交功能
- 商店/道具系统

## 主要改进

1. **性能优化**: 使用配置驱动，减少硬编码
2. **可维护性**: 模块化设计便于维护和调试
3. **扩展性**: 易于添加新功能和关卡
4. **代码复用**: 实体系统支持代码复用
5. **配置管理**: 集中管理游戏参数

## 扩展指南

### 添加新场景
1. 在 `scenes/` 目录下创建新场景文件
2. 继承 `Scene` 基类
3. 在 `main.js` 中注册场景

### 添加新实体
1. 在 `entities/` 目录下创建新实体文件
2. 实现相应的方法
3. 在需要的地方使用新实体

### 添加新配置
1. 在 `GameConfig.js` 中添加新配置项
2. 在相应模块中使用配置值
